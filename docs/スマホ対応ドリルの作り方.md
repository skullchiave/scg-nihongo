# スマホ対応ドリルの作り方（つなぐドリル 共通テンプレ解説）

このドキュメントは `tsunagu/*.html` 全8本に共通する「1ファイルで投影もスマホも両対応」の作りを、**実物のコードから抽出**して言語化したもの。新しいドリルを作るとき／他AIに作らせるときの仕様書として使う。

- 解析対象: `tsunagu/` 8本（L7-1〜L9）。全8本で `@media(max-width:640px)`×2ブロック・`100dvh`・`text-size-adjust`・`.fitwrap`・tap-highlight が完全一致 ＝ 共通テンプレが確立している。
- 大原則: **投影用（PC幅）レイアウトを基準に作り、スマホ幅だけ `@media` で組み替える**。スマホ専用に別ファイルを作らない。1ファイル自己完結・外部依存ゼロ。

---

## 0. なぜこの作りなのか（設計思想）

| 制約 | 採った手 |
|---|---|
| 教室プロジェクター投影と学生スマホ、両方で同じURLを使う | 1ファイルにPC幅とスマホ幅の両レイアウトを内包。`@media(max-width:640px)` で切替 |
| PWA（ホーム画面追加）で使うとブラウザの「戻る」が無い | nav に `🏠ホーム`＋`◀もどる/すすむ▶`（`history.back()/forward()`） |
| iOS Safari が勝手に文字を拡大してカードからはみ出す | `text-size-adjust:100%`（抑止）＋ `.fitwrap` 自動縮小JS（保険） |
| 外部ファイル（音・フォント・JS）を増やしたくない | 効果音は Web Audio で合成。フォントはOS標準。JSは全部インライン |
| スマホは縦長・指タップ、PCは横長・マウス | スマホ幅でナビ2列・選択肢を縦/グリッド・タップ領域拡大 |

## 1. `<head>` の定型（全ドリル共通でコピペ）

```html
<!DOCTYPE html>
<html lang="ja">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<link rel="apple-touch-icon" href="/scg-nihongo/icon-180.png">
<link rel="icon" type="image/png" sizes="192x192" href="/scg-nihongo/icon-192.png">
<link rel="manifest" href="/scg-nihongo/manifest.webmanifest">
<meta name="apple-mobile-web-app-title" content="SCG日本語">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="mobile-web-app-capable" content="yes">
<meta name="theme-color" content="#ffffff">
<title>つなぐ ◯◯ ドリル</title>
```

- `viewport` の `width=device-width` がスマホ対応の出発点。これが無いとPC幅で縮小表示される。
- PWAメタ（apple-/mobile-web-app-*）はホーム画面追加時にアドレスバー無しのアプリ風で開くため。
- アイコン・manifest のパスは GitHub Pages の公開パス（`/scg-nihongo/`）基準の絶対パス。

## 2. スマホ対応の中核CSS（5つの柱）

### 柱① 画面ぴったり＝`100dvh` ＋ flex縦 ＋ `overflow:hidden`

```css
*{box-sizing:border-box;margin:0;padding:0;}
html,body{height:100%;-webkit-text-size-adjust:100%;text-size-adjust:100%;}
body{font-family:"Yu Gothic UI","Yu Gothic","Meiryo",sans-serif;background:var(--bg);color:var(--ink);
  display:flex;flex-direction:column;height:100vh;height:100dvh;overflow:hidden;
  -webkit-text-size-adjust:100%;text-size-adjust:100%;-webkit-user-select:none;user-select:none;}
```

- `height:100vh;height:100dvh;` の**2行重ね書き**が肝。`dvh`（dynamic viewport height）はスマホのアドレスバー出入りで高さが変わる問題に対応。古い端末用に `vh` を先に書いてフォールバックする。
- `overflow:hidden` ＋ `body` を flex 縦 ＝ ナビ固定・本体だけスクロール、の土台。
- `user-select:none` … タップ連打でテキストが選択されてしまうのを防ぐ（学習UIでは選択は不要）。

### 柱② iOS文字自動拡大の抑止＝`text-size-adjust:100%`

```css
html,body{-webkit-text-size-adjust:100%;text-size-adjust:100%;}
```

- iOS Safari は縦画面で文字を勝手に大きくする（font boosting）。これがめくりカードのはみ出し主因。**root（html）に付けるのが効く**。`-webkit-` 付きと無し両方書く。

### 柱③ ナビ＝ホーム＋戻る/進む（PWA対策）

```html
<nav>
  <a class="navhome" href="../">🏠 ホーム</a>
  <button class="navnav" id="navback">◀ もどる</button>
  <button class="navnav" id="navfwd">すすむ ▶</button>
  <span class="ttl">つなぐにほんご L7-2 <small>ない形ドリル</small></span>
  <span class="fzgrp"><button id="fzdown">A−</button><button id="fzval">100%</button><button id="fzup">A＋</button></span>
  <button data-s="rule" class="on">① ルール</button>
  <button data-s="flip">② めくり</button>
  <button data-s="sort">③ グループ分け</button>
  <button data-s="rhythm">④ リズム</button>
</nav>
```
```js
document.getElementById('navback').onclick=()=>history.back();
document.getElementById('navfwd').onclick=()=>history.forward();
```
- `🏠ホーム` は `href="../"`＝母艦ポータル（`index.html`）へ。
- PWA（standalone）はブラウザの戻るボタンが無いので、`◀もどる/すすむ▶`を自前で置く。

### 柱④ 文字サイズ調整＝`zoom`方式（先生が投影中に大きくできる）

```js
let zoom=1;try{zoom=parseFloat(localStorage.getItem('drill_zoom'))||1;}catch(e){}
function applyZoom(){
  zoom=Math.min(1.6,Math.max(0.7,Math.round(zoom*10)/10));
  document.body.style.zoom=zoom;
  document.body.style.height=(100/zoom)+'dvh';   // zoomで縦がはみ出すのを補正
  document.querySelector('nav').style.zoom=1/zoom; // navは打ち消して常に等倍（押しやすさ維持）
  fzval.textContent=Math.round(zoom*100)+'%';
  try{localStorage.setItem('drill_zoom',zoom);}catch(e){}
}
```
- レイアウトごと拡大する `body.style.zoom`。`localStorage` で**全ドリル共通設定**として記憶。
- nav は `zoom:1/zoom` で打ち消し、拡大しても操作ボタンは等倍のまま。
- スマホ幅ではこのボタン群は隠す（後述 `.fzgrp{display:none}`）。

### 柱⑤ 流体サイズ＝`clamp()`（スマホ幅で使う）

PC幅では固定px（投影で大きく見せる）、スマホ幅では `clamp(最小,可変,最大)` で画面幅にフィットさせる。後述の `@media` 内で多用。

## 3. `@media (max-width:640px)` ＝ スマホ組み替え

**ブレークポイントは 640px 固定**。これ以下＝スマホとみなす。投影用CSSはそのまま、スマホだけ上書きする。

### 3-1. 主ブロック（共通の組み替え）

```css
@media (max-width:640px){
  body{height:100dvh;}
  /* ナビ: タイトル全幅 → ボタンは2×2 */
  nav .ttl{font-size:14px;width:100%;margin:0 0 2px;}
  nav .fzgrp{display:none;}                       /* 文字サイズボタンはスマホで隠す */
  nav button{flex:1 1 calc(50% - 3px);min-width:0;padding:11px 4px;font-size:14px;} /* 2列折返し */
  .screen{padding:14px 12px;}
  h2{font-size:19px;line-height:1.35;}
  h2 span{display:block;font-size:13px!important;margin-top:3px;} /* 副題を改行 */
  /* ① ルール: 3カラム → 1カラム縦積み */
  .rules{grid-template-columns:1fr;gap:12px;}
  /* ② めくり: 自動カラム → 2カラム固定。文字は clamp */
  .grid{grid-template-columns:1fr 1fr;gap:10px;}
  .fcard{height:94px;}
  .front,.back{font-size:clamp(15px,5vw,21px);}
  /* ③ 並べ替え: カード全幅・選択肢を縦並び・タップ領域大 */
  .qcard{font-size:clamp(28px,9.5vw,40px);min-width:0;width:100%;max-width:92vw;padding:22px 14px;}
  .choices{flex-direction:column;width:100%;max-width:340px;gap:12px;}
  .choices button{width:100%;font-size:18px;padding:15px;}
  .score{position:static;align-self:flex-end;}   /* 絶対配置 → 通常フローへ */
  /* ④ リズム */
  .rcard{font-size:clamp(34px,10.5vw,72px);}
}
```

### 3-2. 第2ブロック（③並べ替えを「1画面に収める＝スクロールなし」専用調整）

スマホで③だけはスクロールさせたくない（テンポが切れる）ので、`#sort` に限定してさらに詰める。

```css
@media (max-width:640px){
  #sort .sortwrap{gap:7px;justify-content:space-evenly;padding-top:2px;}
  #sort .qcard{font-size:clamp(22px,6.5vw,32px);padding:10px 12px;}
  #sort .choices{display:grid;grid-template-columns:repeat(auto-fit,minmax(96px,1fr));gap:7px;max-width:360px;}
  #sort .choices button{width:auto;padding:11px 6px;font-size:15px;white-space:nowrap;}
  #sort .score{position:absolute;top:10px;right:14px;}
}
```
- ポイント: 「全画面共通の組み替え」と「特定画面の追い込み」を**2ブロックに分ける**。後者は `#sort` 等のIDで限定。

## 4. タップ操作の作法（スマホで気持ちよく押せる）

```css
.catbtn, .navhome, .choices button { -webkit-tap-highlight-color:transparent; }
.catbtn:active{transform:scale(.985);}
.choices button:active{transform:scale(.96);}
```
- `-webkit-tap-highlight-color:transparent` … タップ時の青いハイライトを消す（アプリ感）。
- `:active{transform:scale(.97前後)}` … 押した瞬間に少し縮む＝押した感のフィードバック。hover に頼らない（スマホに hover は無い）。

## 5. はみ出し保険＝`.fitwrap` 自動縮小JS

`text-size-adjust` で抑えきれない長い語やiOSの拡大で、めくりカードから文字がはみ出す時、**JSで検知して `transform:scale` で縮める**保険。flipのDOM生成（flipDeal）は触らず後付けで動く汎用処理。

```css
.fitwrap{display:flex;flex-direction:column;align-items:center;justify-content:center;white-space:nowrap;transform-origin:center;}
```
```js
(function(){
  var grid=document.getElementById('flipgrid'); if(!grid) return;
  function fitOne(face){
    var wrap=face.querySelector(':scope > .fitwrap');
    if(!wrap){ wrap=document.createElement('span'); wrap.className='fitwrap';
      while(face.firstChild) wrap.appendChild(face.firstChild); face.appendChild(wrap); }
    wrap.style.transform='';
    var cw=face.clientWidth, ch=face.clientHeight, sw=wrap.scrollWidth, sh=wrap.scrollHeight;
    if(!cw||!sw) return;                 // 非表示中(0px)はスキップ
    var s=Math.min(cw/sw, ch/sh, 1);
    if(s<0.99) wrap.style.transform='scale('+(s*0.96).toFixed(3)+')';
  }
  function fit(){ requestAnimationFrame(function(){ grid.querySelectorAll('.face').forEach(fitOne); }); }
  new MutationObserver(fit).observe(grid,{childList:true});   // カード再生成を検知
  if('IntersectionObserver' in window){                       // 表示された瞬間に再計算
    new IntersectionObserver(function(es){ if(es.some(e=>e.isIntersecting)) fit(); }).observe(grid);
  }
  window.addEventListener('resize',fit);
  window.addEventListener('orientationchange',fit);           // 画面回転時
  fit();
})();
```
- 非表示タブ（width=0）はスキップ。表示・回転・リサイズ・カード再生成すべてで再フィット。
- **iOS実機はこの開発環境で再現できない → 最終確認はユーザーのiPhoneで**。

## 6. 4モード構成（ドリル本体の中身）

全ドリル共通の4タブ。`show(id)` でセクション表示を切替（タブ切替は履歴に積まない）。

| # | id | 名前 | 中身 |
|---|---|---|---|
| ① | `rule` | ルール | 文法の作り方を3カラム（スマホ1カラム）で説明 |
| ② | `flip` | めくり | カードをタップで裏返して答え合わせ（`flipDeal()` で動的生成・シャッフル） |
| ③ | `sort` | グループ分け等 | 出題→3択タップ→正誤フィードバック＋スコア。1画面に収める |
| ④ | `rhythm` | リズム | 画面タップ/Spaceで「ます形→ない形」をテンポよく変換 |

```js
const btns=document.querySelectorAll('nav button[data-s]');
const screens=document.querySelectorAll('.screen');
function show(id){
  screens.forEach(s=>s.classList.toggle('on',s.id===id));
  btns.forEach(b=>b.classList.toggle('on',b.dataset.s===id));
}
btns.forEach(b=>b.onclick=()=>show(b.dataset.s));
// キーボード: 1〜4でタブ切替、Space/→で次へ（投影中の操作用）
```

- 出題語は冒頭の `const VERBS=[{m,d,n,g,ex},...]`（ます形/辞書形/ない形/グループ/メモ）。**ここを差し替えれば別の課のドリルになる**。
- 効果音は Web Audio 合成（`sndOK`＝ピンポーン / `sndNG`＝ブザー）。外部音源ファイル不要。

## 7. 新しいドリルを作る手順（チェックリスト）

1. 既存の近いドリル（例 `tsunagu/L7-2.html`）を**丸ごとコピー**して名前変更。
2. `<title>` と nav の `.ttl` を直す。
3. `const VERBS=[...]`（出題データ）を新しい課の語に差し替える。
4. ① ルール画面の説明文（3カラム）を文法に合わせて書き換え。
5. PC幅（ブラウザを広く）で投影レイアウト確認 → **ウィンドウを 640px 以下に狭めて**スマホレイアウト確認。
6. 4タブ（特に③が1画面に収まるか）と、②めくりの長い語のはみ出しを確認。
7. iOS実機（自分のiPhone）でめくりカードのはみ出し最終確認。
8. `index.html` 冒頭の `DATA` に1行追加（`ready:true`）→ `git add/commit/push`。

> 検証の自動化（ヘッドレスChrome＋CDPでスクショ・DOM計測）の手順は `CLAUDE.md` の「検証手順」参照。試作・スクショは `scratch/` に閉じ込める。

## 8. やらない・触らないルール（事故防止）

- `index.html` は**手編集が正本**。`scratch/_RETIRED_build_index.py` は**絶対に実行しない**（手編集が消える）。
